プルに失敗する原因と対応¶

プルに失敗したときの原因と対応方法です。

● 原因と対応
● 失敗と対応の例 #1 ： コンフリクトなし
　　・ ファイルを編集
　　・ プル　→　エラー発生
　　・ コミット
　　・ 再プル
　　・ プルしたファイルの確認
● 失敗と対応の例 #2 ： コンフリクトあり
　　・ ファイルを編集
　　・ プル　→　エラー発生
　　・ コミット
　　・ 再プル　→　コンフリクト発生
　　・ コンフリクトが発生したファイルの修正

原因と対応¶

原因

  • 未コミットの更新したファイルがプルにより更新されるファイルとかち合ったため。

  • かち合った未コミットのファイルをプルで書き換えると更新内容がなくなる可能性があります。そのため、未コミットの更新したファイルがプルで更新する対象とかち合った場合にプルは失敗します。

対応

次のいずれかの対応後、再度「プル」します。

  • かち合ったファイルをコミットする

  • かち合ったファイルを削除する

  • かち合ったファイルを他のフォルダーへ移動する

失敗と対応の例 #1 ： コンフリクトなし¶

ファイルを編集¶

index.rst ファイル内の「経緯」を変更します。変更後セーブしますが、コミットはしません。

↓


プル　→　エラー発生¶

プル対象ファイルと未コミットのファイルがかち合うためエラーが発生します。

  1. リモートリポジトリの状態を確認

    コミット状況の確認です。自分（ riki ）がプッシュ後に新しい内容が追加されています。


    詳細です。 docs/source/index.rst がプッシュされています。


  2. プル をクリック


  3. プル元を確認　→　プル をクリック


  4. プル中にエラー発生　→　閉じる(C) をクリック


    エラーメッセージと翻訳結果です。

    Your local changes to the following files would be overwritten by merge:

    docs/source/index.rst

    Please commit your changes or stash them before you merge.

    ↓

    以下のファイルに対するローカルでの変更は、マージによって上書きされます。

    docs/source/index.rst

    マージする前に変更をコミットするか、隠しておいてください。


  5. キャンセル をクリック


  6. プル終了

    プル中にエラーが発生したため、リモートリポジトリの内容は取り込まれていません。


コミット¶

未コミットのファイルをコミットします。

  1. コミット前の状態　→　コミットされていない変更があります をクリック


  2. 対象ファイルをステージングエリアに登録　→　コミット をクリック


  3. コメントを入力　→　コミット をクリック


  4. コミット終了


再プル¶

  1. プル をクリック


  2. プル元を確認　→　プル をクリック


  3. プル終了


プルしたファイルの確認¶

最初に変更した部分（赤枠）とプルした部分（青枠）がマージされました。


失敗と対応の例 #2 ： コンフリクトあり¶

ファイルを編集¶

conf.py ファイル内の「project=」を変更します。変更後セーブしますが、コミットはしません。

↓


プル　→　エラー発生¶

プル対象ファイルと未コミットのファイルがかち合うためエラーが発生します。

  1. リモートリポジトリの状態を確認

    コミット状況の確認です。自分（ riki ）がプッシュ後に新しい内容が追加されています。


    詳細です。 docs/source/conf.py がプッシュされています。


  2. プル をクリック


  3. プル元を確認　→　プル をクリック


  4. プル中にエラー発生　→　閉じる(C) をクリック


    エラーメッセージと翻訳結果です。

    Your local changes to the following files would be overwritten by merge:

    docs/source/conf.py

    Please commit your changes or stash them before you merge.

    ↓

    以下のファイルに対するローカルでの変更は、マージによって上書きされます。

    docs/source/conf.py

    マージする前に変更をコミットするか、隠しておいてください。


  5. キャンセル をクリック


  6. プル終了

    プル中にエラーが発生したため、リモートリポジトリの内容は取り込まれていません。


コミット¶

未コミットのファイルをコミットします。

  1. コミット前の状態　→　コミットされていない変更があります をクリック


  2. 対象ファイルをステージングエリアに登録　→　コミット をクリック


  3. コメントを入力　→　コミット をクリック


  4. コミット終了


再プル　→　コンフリクト発生¶

コンフリクト（衝突）

同じファイルの同じ箇所の修正がかち合った（コンフリクト、衝突）状態のこと。

  1. プル をクリック


  2. プル元を確認　→　プル をクリック


  3. プル中にコンフリクトが発生　→　閉じる(C) をクリック


  4. プル終了

    プル中にコンフリクトが発生しましたが、ローカルリポジトリにリモートリポジトリの変更内容が取り込まれました。 ！ マークが付いているのがコンフリクトが発生したファイルです。


コンフリクトが発生したファイルの修正¶

  1. コンフリクトが発生した docs/source/conf.py ファイルを開く

    コンフリクトマーカー（衝突マーカ－）の確認　＝　コンフリクトが発生した部分が対象ごとに色分けして編みかけされています。


  2. 対応方法の候補から適切なものを選択

    次の 3 つの対応方法の候補が表示されています。
    ・ 現在の変更を取り込む　＝　ローカルリポジトリの内容を取り込みます
    ・ 入力側の変更を取り込む　＝　リモートリポジトリの内容を取り込みます
    ・ 両方の変更を取り込む　＝　両リポジトリの内容を取り込みます
    今回は「現在の変更を取り込む」を選択します。

  3. 修正内容の保存

    選択した候補の内容だけが表示されています。他に修正が必要な箇所があれば、このタイミングで修正します。修正後はファイルをセーブします。
