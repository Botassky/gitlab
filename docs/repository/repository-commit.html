コミット¶

ワーキングディレクトリ内にいくつかのフォルダーやファイルを作成したので、その内容をコミットします。

● コミット前の状態
● Sourcetree
● コミット後の状態

コミット前の状態¶

ファイル作成前（クローン直後）の状態です。クローン直後なので、 master ・ origin/master ・ origin/HEAD が同じ位置です。


Sourcetree¶

  1. History をクリック


  2. コミットされていない変更があります をクリック


  3. 作業ツリーのファイル にワーキングディレクトリで追加・削除・変更などがあったファイルが表示されていることを確認


  4. 作業ツリーのファイル 内からコミットするファイルを Index にステージしたファイル に移動（今回は 全てインデックスに追加 をクリックして全てのファイルを移動した）


    Tip

    ファイルの移動は次のいずれかの方法で行います。

    • ドラッグ＆ドロップ

    • ファイル名（パス）の右側の ＋ をクリック

    • ファイルを選択（ファイル名をクリック）後、 選択をインデックスに追加 をクリック

    • 全てインデックスに追加 をクリック


  5. コミット をクリック


  6. コミット欄にコミットする内容を記述


  7. コミット をクリック


  8. コミット終了


  9. History をクリック　→　コミットの内容を確認


コミット後の状態¶

コミットの実行で履歴が追加され、 master が履歴の先頭を指しています。 origin/master と origin/HEAD は変わらないため、コミット前の位置のままです。
