プッシュ¶

コミット後のローカルリポジトリの内容をリモートリポジトリに反映します。反映対象はコミットしたファイルです。

プッシュ前の状態¶

GitLab サーバー¶

クローン 時の状態と同じです。

Sourcetree¶

コミット後の状態 と同じです。


Sourcetree¶

  1. プッシュ をクリック


  2. 次のようになっていることを確認　→　プッシュ をクリック

    項目

    選択値

    備考

    対象

    チェックが入っていること

    ローカルブランチ

    master

    リモートブランチ

    master

  3. プッシュ中


  4. プッシュ終了


プッシュ後の状態¶

GitLab サーバー¶

コミット したフォルダーやファイルがプッシュ（アップロード）され、ローカルリポジトリの内容がリモートリポジトリに反映されました。


doc フォルダーの内容です。


source フォルダーの内容です。

Sourcetree¶

ローカルリポジトリとリモートリポジトリが同じ内容になったので、 master ・ origin/master ・ origin/HEAD が同じ位置に表示されています。


履歴の確認¶

GitLab サーバー¶

  1. プロジェクト名をクリック


  2. 2 Commits をクリック

    “2 Commits” は 2 回コミットされたという意味です。


  3. コミットの履歴を確認


  4. コメントをクリック


  5. コミット内容の詳細を表示

Sourcetree¶

  1. History をクリック


  2. コミット内容を確認
