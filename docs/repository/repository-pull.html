プル¶

リモートリポジトリの変更をローカルリポジトリに取り込みます。

● プル前の状態
● Sourcetree
● プル後の状態

プル前の状態¶

GitLab サーバー¶

リモートリポジトリのコミットの履歴を確認すると、自分（ riki ）がリモートリポジトリにプッシュした後に jiro がプッシュしています。今回はこの jiro がプッシュした内容をローカルリポジトリに取り込みます。

Sourcetree¶

プッシュ後の状態 と同じです。


Sourcetree¶

  1. プル をクリック


  2. 次のようになっていることを確認　→　プル をクリック

    項目

    選択値

    備考

    次のリモートからプル

    origin

    「リモートリポジトリ」のこと

    プルするリモートブランチ

    master

    次のブランチにプル

    master

  3. プル中


  4. プル終了


プル後の状態¶

GitLab サーバー¶

プルはリモートリポジトリの内容を更新しないので プル前の状態 と同じです。

Sourcetree¶

jiro がコミットした内容を取り込み、履歴が進んでいます。プル直後はリモートリポジトリとローカルリポジトリが同期されてているので master ・ origin/master ・ origin/HEAD は同じ位置です。

取り込まれたファイルは以下のとおりです。
・ 修正　docs/source/conf.py
・ 修正　docs/source/index.rst
・ 追加　docs/source/japanesesupport.py
