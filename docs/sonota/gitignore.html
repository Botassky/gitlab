ステージングエリアから除外するファイルの指定¶

ビルドなどで作成できるファイルは版（バージョン）管理をする必要はありません。これらのファイルをステージングエリアに登録しなければ良いのですが、都度登録しないよう操作したり、間違って登録したものを除外するのは手間がかかります。このように Git の管理から除外したいファイルの指定に .gitignore ファイルを使用します。 .gitignore ファイルに除外したいファイルやフォルダーを記録すると、それらはステージングエリアへの登録から除外されます。

Tip

  • 「ステージングエリアに登録しない　＝　コミットしない　＝　リモートリポジトリに登録しない」になります。

  • .gitignore ファイルに指定した内容はプロジェクト全体に影響します。

  • .gitignore ファイルに指定する前に 1 回でもコミットしたファイルは .gitignore ファイルに指定しても、その指定は無視されます。

ルール¶

基本ルール¶

  • .gitignore ファイルを置いたフォルダーおよびそれよりも下位のフォルダー内のファイルが対象になります。

  • ファイル名を指定すると、そのファイルを無視します。指定したファイルが複数のフォルダー内に存在する場合、そのすべてを無視します。

  • フォルダーは .gitignore ファイルを置いたフォルダーをカレントとした相対パスで指定します。

  • フォルダーを無視するときはフォルダー名の最後に “/” を付けます。この場合、指定したフォルダーに含まれるサブフォルダーも無視されます。

  • “!” で始まる行は、それ以前の無視を打ち消します（無視を無効にします）。

  • “#” で始まる行はコメント扱いです。

  • 空行は何もしません。

その他のルール¶

  • * は任意の 0 文字以上の文字列として扱います。

  • ? は任意の 1 文字として扱います。

  • [Dd] のように特定の 1 文字（この場合は “D” または “d” ）の指定ができます。

  • [0-9] のように範囲の含む特定の 1 文字（この場合は “0”, “1”, “2”, ・・・ “8”, “9” ）の指定ができます。

サンプル¶

” https://github.com/github/gitignore “に多数の :file:”.gitignore” のサンプルが登録されています。 .gitignore ファイルを一から作成するのではなく、これらのサンプルを活用または参考にするほうが良いでしょう。

以下のコードは上記のサンプルの組み合わせに、少しカスタマイズしたものです。

# Windows thumbnail cache files
Thumbs.db
Thumbs.db:encryptable
ehthumbs.db
ehthumbs_vista.db

# Dump file
*.stackdump

# Folder config file
[Dd]esktop.ini

# Recycle Bin used on file shares
$RECYCLE.BIN/

# Windows Installer files
*.cab
*.msi
*.msix
*.msm
*.msp

# Windows shortcuts
*.lnk

# VisualStudioCode.gitignore
.vscode/*
!.vscode/settings.json
!.vscode/tasks.json
!.vscode/launch.json
!.vscode/extensions.json
*.code-workspace

# Local History for Visual Studio Code
.history/

# Sphinx build file
docs/build/
docs/_build/

設定（作成～プッシュまで）¶

Sourcetree で設定～プッシュまで行います。

  1. 設定 をクリック


  2. 詳細 をクリック


  3. 編集 をクリック


  4. 使用しているエディタを選択（今回は Visual Studio Code を選択）　→　OK をクリック


  5. .gitignore ファイルの編集前の状態（何も記載されていません）


  6. サンプル の内容を記載　→　セーブ　→　エディタを終了


  7. OK をクリック


  8. .gitignore ファイルをステージングエリアに移動


  9. .gitignore ファイルがステージングエリアに移動したことを確認　→　コミット をクリック


  10. コミット時のメッセージを記入　→　コミット をクリック


  11. コミット終了　→　プッシュ をクリック


  12. 「ローカルブランチ」と「リモートブランチ」を確認　→　プッシュ をクリック


  13. プッシュ中


  14. プッシュ終了


  15. History をクリック　→　履歴を確認


GitLab サーバーの状態¶

プッシュした .gitignore ファイルを確認します。

  1. .gitignore ファイルがアップロードされていることを確認


  2. .gitignore ファイルの内容を確認


“.gitignore” ファイルの取り込み¶

リモートリポジトリに .gitignore ファイルが登録されているときはプルします。

  1. プル をクリック


  2. プル元／プル先を確認　→　プル をクリック


  3. プル中


  4. プル終了


  5. 設定 をクリック


  6. 詳細 をクリック


  7. 編集 をクリック


  8. プルした .gitignore ファイルの内容を確認　→　エディタを閉じる


  9. キャンセル をクリック　→　終了


コミット済みファイルを除外する¶

.gitignore ファイルに記入する前にコミットしたファイルは、ファイルに記入しても除外になりません。そのため、手動で除外設定が必要です。

docs/source/memo.txt をコミットした後に .ignore ファイルに追加しました。

.gitignore ファイルに記載する前にコミットしたため、編集するとコミット候補として表示されます。

  1. 除外するファイルを選択


  2. 選択したファイルを右クリック　→　追跡をやめる をクリック


  3. OK をクリック


  4. 除外終了


  5. コミット をクリック


  6. コメントを入力　→　コミット をクリック


  7. コミット終了　→　プッシュ をクリック


  8. プッシュするブランチを確認　→　プッシュ をクリック


  9. プッシュ終了（＝　リモートリポジトリ上の除外ファイルの削除が終了）


フォルダーの場合

ファイルの場合は上記方法で対応できますが、フォルダーの場合は下記の手順になります。

  1. 対象フォルダーを Git の管理外へ移動

  2. .gitignore ファイルから、当該フォルダーの行を削除

  3. 「コミット　→　プッシュ」でリモートリポジトリから対象フォルダーを削除

  4. .gitignore ファイルに当該フォルダーの行を追加

  5. 「コミット　→　プッシュ」で .gitignore ファイルをリモートリポジトリに登録
