差分確認¶

コミットを 2 箇所選択すると、コミット間の差分をファイル単位で確認できます。

抽出されたファイルを選択すると、ファイル内の差異を表示します。


過去のコミットから作業のやり直し¶

過去に戻って作業をやり直したいときは、過去のコミットから新規ブランチを作成します。作成した新規ブランチで作業します。

  1. 戻りたいコミットをダブルクリック


  2. OK をクリック


  3. HEAD がダブルクリックしたコミットに表示されたことを確認

    Note

    1 つ前のステップのメッセージで表示されたようにこの HEAD はどのブランチにも属していません。


  4. ブランチ をクリック


  5. 新規ブランチ にブランチ名を入力　→　ブランチを作成 をクリック


  6. 新しいブランチの作成終了

    新しく作成した edit ブランチは元になったコミット以降の変更は反映されていません（＝　過去に戻ったのと同じ状態）。


edit ブランチで必要な作業が終了したら master ブランチに マージ します。下図は edit ブランチで 1 回コミットしたあとに master ブランチにマージした状態です。

リバート（過去のコミットを打ち消す）¶

特定のコミット内容と逆のコミット内容を作り出すことでコミットを打ち消します。

コンフリクトなし¶

打ち消し前の状態です。コミット内容は次のとおりです。
・ index.rst ファイルの更新
・ otoiawase.rst ファイルの追加

  1. 打ち消し対象のコミットを右クリック　→　このコミットを打ち消し… をクリック


  2. はい をクリック


  3. 打ち消し中


  4. 打ち消し終了


打ち消し後の状態です。次の内容のコミットが追加されました。
・ index.rst ファイルの修正
・ otoiawase.rst ファイルの削除

この状態でプッシュすると、リモートリポジトリでもコミットが打ち消されます。


コンフリクトあり¶

打ち消し前の状態です。コミット内容は次のとおりです。
・ index.rst ファイルの更新

今回は打ち消し対象が他のコミットの変更との関係でコンフリクトが発生します。


  1. 打ち消し対象のコミットを右クリック　→　このコミットを打ち消し… をクリック


  2. はい をクリック


  3. 打ち消し中


  4. コンフリクト発生　→　閉じる をクリック


  5. 打ち消し終了

    コンフリクトが発生したため、リバート後の内容でコミットは作成されませんでした。


  6. コンフリクトが発生した index.rst ファイルを開く　→　コンフリクトが発生した箇所を確認


  7. 選択肢の中から適切な動作を選択（今回は 入力側の変更を取り込む を選択）


  8. 必要ならこのタイミングで修正　→　ファイルを保存


  9. index.rst ファイルをステージングエリアに登録

    ↓


  10. コミット をクリック


  11. コミットのコメントを入力　→　コミット をクリック


  12. 終了


この状態でプッシュすると、リモートリポジトリでもコミットが打ち消されます。


チェリーピック（他のブランチのコミットを取り込む）¶

他のブランチのコミットを現在のブランチに取り込みます。チェリーピック後も取込み元のコミットはそのままです。今回は master ブランチのコミットを edit ブランチに取り込みます。

コンフリクトなし¶

  1. 編集対象が edit ブランチであることを確認


  2. master ブランチから取り込むコミットを右クリック　→　チェリーピック をクリック


  3. OK をクリック


    【英文　→　和訳】

    • Cherry picking lets you select a commit and apply only those changes to your current branch.

      チェリーピッキングでは、コミットを選択して現在のブランチに変更点のみを適用することができます。

    • Commit immediately upon successful merge.

      マージに成功したらすぐにコミットします。

    • Include commit IDs in the merge commit message.

      マージコミットメッセージにコミット ID を含めます。


  4. チェリーピック終了

    master ブランチのコミットの取込みが終了しました。コミットメッセージは master ブランチの値をそのまま使用しています。


コンフリクトあり¶

  1. 編集対象が edit ブランチであることを確認


  2. master ブランチから取り込むコミットを右クリック　→　チェリーピック をクリック


  3. OK をクリック


    【英文　→　和訳】

    • Cherry picking lets you select a commit and apply only those changes to your current branch.

      チェリーピッキングでは、コミットを選択して現在のブランチに変更点のみを適用することができます。

    • Commit immediately upon successful merge.

      マージに成功したらすぐにコミットします。

    • Include commit IDs in the merge commit message.

      マージコミットメッセージにコミット ID を含めます。


  4. コンフリクト発生　→　閉じる (C) をクリック


  5. チェリーピック終了

    コンフリクトが発生したため、チェリーピック後の内容でコミットは作成されませんでした。


  6. コンフリクトが発生した staff.rst ファイルを開く　→　コンフリクトが発生した箇所を確認


  7. 選択肢の中から適切な動作を選択（今回は 両方の変更を取り込む を選択）


  8. 必要ならこのタイミングで修正　→　ファイルを保存


  9. index.rst ファイルをステージングエリアに登録

    ↓


  10. コミット をクリック


  11. コミットのコメントを入力　→　コミット をクリック


  12. 終了


  13. コミットの履歴に追加されていることを確認


最後のコミットメッセージの修正¶

プッシュ前であれば最後（直近）のコミットメッセージを修正できます。

  1. コミット をクリック


  2. オプションのコミット のプルダウンリスト内から 最後のコミットを上書き (Amend) をクリック


  3. はい をクリック


  4. コミットメッセージ欄に最後（直近）のコミットメッセージが表示されたことを確認


  5. コミットメッセージを修正　→　コミット をクリック


  6. 最後（直近）のコミットメッセージの修正終了


  7. 修正結果の確認


コミットにタグを付ける／外す¶

コミットに「リリース版」や「 v1.0 」等のタグを付けられます。付けたタグは外せます。

タグを付ける¶

  1. タグを付けるコミットを右クリック　→　タグ… をクリック


  2. タグ名 にタグの名前を入力　→　タグを追加 をクリック


  3. タグ付け終了


左側の タグ に設定したタグの一覧が表示されます。下図は 5 個のタグを設定した結果です。


Tip

タグの名前を / で前半分、後ろ半分に区切ると、上図のように前半分でグルーピングできます。

タグを外す¶

今回は “リリース版/v1.1” のタグを削除します。

  1. タグ をクリック


  2. タグを削除 をクリック


  3. 削除するタグ をプルダウン　→　削除するタグを選択


  4. タグを削除 をクリック


  5. タグの削除終了


タグをプッシュ¶

ブランチをプッシュしてもタグはプッシュされません。設定したタグごとにプッシュしなければなりません。

GitLab で現状のタグを確認¶

  1. プロジェクト名をクリック


  2. Repository 内の Tags をクリック


  3. 現在のタグを表示

    まだタグが 1 つもプッシュされていません。


SourceTree¶

  1. プッシュするタグを右クリック　→　プッシュ先 をクリック　→　origin をクリック


  2. OK をクリック


  3. タグをプッシュ中


  4. プッシュ終了


GitLab でプッシュしたタグを確認¶

プッシュしたタグが表示されていることを確認できます。
